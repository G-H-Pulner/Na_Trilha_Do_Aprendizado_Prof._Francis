O que é uma Branche?

Uma Branche é uma extensão do seu projeto inicial, ela pode ser uma nova funcionalidade, atualização, correção de erro ou qualquer coisa que necessite ser desenvolvida separadamente do projeto principal por conta do site não poder ficar em constante mudança (um site no ar não pode ficar sofrendo atualizações constantes), um site que não precise disto não precisa de uma Branche, pois não faz mal algum sofrer constantes atualizações, um bom exemplo de implementação de Branches é no caso de um desenvolvimento em equipe, assim cada um desenvolve uma parte do site separadamente para assim que finalizado ser mesclado com a Branche master ou main. Todas as Branches são feitas a partir do master.


Como fazer uma branche

Primeiro deve-se criar um repositório, depois desenvolva a Branche main ou master (isso deve ser feito pois a Branche posterior é apenas uma adição à Branche principal), veja o video a seguir para aprender o processo necessário para a criação de uma Branche. VIDEO (O video foi feito com base no GitHub Desktop)

Passo a Passo no GitHub desktop no 1º modo de se fazer uma Branche: Current branche → New branche → Coloca o nome da branche → Create branche → Publish branche.


Como juntar as branches

Acesse o VIDEO para saber como juntar as branches, se juntar uma branche com o main, os conteúdos serão mesclados, poderá dar erro caso junte um main com uma branche incompatível, esse erro é um erro de compatibilidade da branche com a main, para resolver o problema é necessário escolher uma das opções que estão disponíveis no VS Code, é possível também fazer branches baseadas em outras branche e atualizar a branche para o mesmo conteúdo do main. O nome da mesclagem é merge.


Hospedagem

Para conseguir hospedar um site no GitHub Pages, é necessário que seu site esteja publico e não seja comercial, isso deixa bem restrito a hospedagem mas é de graça e dá uma noção de como é uma hospedagem num serviço de hospedagem pago.


Como hospedar um site

Selecione o repositório → Settings → Desça até GitHub Pages e acesse o Check it out here! → vá em Source e mude o None para Main → Salve as configurações em Save. Você receberá um link com seu site já hospedado, veja ele no GitHub Pages.

Observações: Você deve deixar o index.html livre, não coloque ele em nenhuma pasta, se ele for colocado em alguma pasta o sistema não conseguira achar e apenas hospedara o README.
